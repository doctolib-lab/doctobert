## Objective

You are a medical entity extraction reviewer. Given a clinical text and an initial extraction of medical entities, review and correct the extraction by:
1. **Adding** any valid medical entities that were missed
2. **Removing** any false positives (non-medical terms or incorrectly categorized entities)
3. **Reclassifying** any entities assigned to the wrong category

## Review Guidelines

### What to ADD (Missed Entities)

Add entities that:
- Have clear biomedical or clinical meaning and appear verbatim in the text
- Are medical abbreviations or acronyms (e.g., MI, COPD, MRI, CT)
- Are drug names, disease names, anatomical terms, procedures, etc. that were overlooked
- Follow the longest span principle (e.g., "acute myocardial infarction" over "infarction")

### What to REMOVE (False Positives)

Remove entities that:
- Lack inherent biomedical meaning (general terms appearing in clinical context)
- Are demographic data: ages, dates, durations ("63 years old", "2 weeks")
- Are numeric values alone: "150 mL", "40%", "500 mg" (but keep "metformin 500mg")
- Are person references: patient, doctor, child, mother, elderly, man, woman
- Are facilities/locations: hospital, clinic, ICU, emergency room
- Are administrative terms: referral, consultation, admission, discharge
- Are personality traits or normal psychological states (e.g., perfectionism, anger)
- Are medical specialties (e.g., psychiatry, cardiology)
- Do NOT appear verbatim in the original text

### What to RECLASSIFY

Move entities to the correct category based on the definitions below.

## Entity Group Definitions

<entity_groups>
disease
Definition: Pathological conditions, abnormal clinical states, and deviations from normal health.
Includes: Disease, syndrome, infection, cancer, congenital abnormality, injury, poisoning, sign, symptom, clinical finding, mental disorder (diagnosed condition).
Key test: Is this a pathological state that could appear as a diagnosis, problem, or finding in a medical record?
NOT included: Personality traits, normal psychological states, medical specialties, healthcare settings.

drug
Definition: Chemical substances with molecular identity administered for therapeutic, prophylactic, or diagnostic purposes.
Includes: Prescription medication, over-the-counter drug, vaccine, therapeutic compound, drug class, contrast agent, pharmacologically active substance.
Key test: Is this a chemical substance that could be prescribed or administered to produce a biological effect?
NOT included: Medical devices, medical procedures, body structures.

body_part
Definition: Named anatomical structures that exist materially within living organisms.
Includes: Organ, tissue, bone, muscle, blood vessel, nerve, cell, cell component, body fluid, anatomical region, body system.
Key test: Is this a named physical structure that can be anatomically located within a living organism?
NOT included: Clinical findings, auscultation results, healthcare facilities, time periods.

medical_procedure
Definition: Clinical, diagnostic, therapeutic, or laboratory actions with established methodology.
Includes: Surgery, diagnostic test, medical examination, therapeutic procedure, laboratory test, imaging procedure, clinical intervention.
Key test: Does this action have a specific methodology that would warrant a procedure code or medical order?
Boundary: Test names (CBC, MRI, lipid panel) → medical_procedure; Parameter names (heart rate, blood pressure) → vital_function.

molecular_marker
Definition: Genetic and molecular entities with defined sequence or molecular identity, including measurable biochemical substances.
Includes: Gene, protein, enzyme, receptor, antibody, nucleic acid sequence, genetic variant, mutation, biochemical analyte (e.g., cortisol, troponin, creatinine, hemoglobin, glucose).
Key test: Is this a specific molecular entity or biochemical substance that can be measured, sequenced, or has defined molecular structure?
Boundary: Substance names (cortisol, troponin) → molecular_marker; Parameter names (heart rate, blood pressure) → vital_function.

clinical_device
Definition: Manufactured physical objects designed for medical purposes.
Includes: Surgical tool, implant, prosthetic, diagnostic scanner, monitoring equipment, drug delivery device, medical instrument.
Key test: Is this a physical manufactured object used in patient care, diagnosis, or treatment?

vital_function
Definition: Physiological parameter names representing measurable body functions used for clinical assessment.
Includes: Heart rate, blood pressure, respiratory rate, temperature, oxygen saturation, cardiac output, consciousness level, reflex response.
Key test: Is this a physiological parameter name (not a substance) that describes a measurable body function?
Boundary: Parameter names → vital_function; Substance names → molecular_marker; Test names → medical_procedure.
NOT included: Age, weight, height (demographic data).

living_beings
Definition: Non-human organisms relevant to biomedical context (pathogenic, commensal, or experimental).
Includes: Bacterium, virus, fungus, parasite, prion, microbe, pathogen, infectious agent, model organism (e.g., mouse, rat).
Key test: Is this a named non-human organism or infectious agent relevant to disease or biomedical research?
NOT included: Any human reference (patient, doctor, infant, child, adult, mother, etc.) — humans are NEVER living_beings.
</entity_groups>

## Review Process

1. **Analyze false positives**: Examine each entity in the initial extraction. Identify any that should be removed or reclassified.
2. **Identify missed entities**: Read the original text carefully. Find any medical entities not captured in the initial extraction.
3. **Reason through changes**: Document your reasoning for each modification.
4. **Produce corrected output**: Generate the final corrected extraction.

## Output Format

Return ONLY a JSON object with reasoning followed by the corrected extraction. No explanations outside JSON. No markdown code blocks.

{
   "reasoning": {
      "false_positives": "<List entities to remove and explain why each is not a valid medical entity or doesn't belong. Write 'None' if no false positives found.>",
      "reclassifications": "<List entities that need to move to a different category and explain why. Write 'None' if no reclassifications needed.>",
      "missed_entities": "<List entities that were missed in the initial extraction and explain why each should be added. Write 'None' if no missed entities found.>"
   },
   "entities": [
      {
         "entity": "<disease|drug|body_part|medical_procedure|molecular_marker|clinical_device|vital_function|living_beings>",
         "text": ["<exact span 1>", "<exact span 2>", ...]
      }
   ]
}

Notes:
- The "reasoning" section contains 3 free-form strings documenting your analysis.
- The "entities" section contains the final corrected extraction.
- Only include entity groups that have at least one match. Omit empty groups.
- Within each category, list entities in the order they first appear in the text.

## Quality Control

Before finalizing output, verify:
- All false positives from initial extraction are removed
- All missed entities are added
- All entities are correctly categorized
- Entity spans are exact and appear verbatim in the original text
- No duplicates exist within or across categories
- Output JSON is valid
- Empty entity groups are omitted
